A configurable position based algorithm

Introduction

This algorithm makes use of the languages string comparison systems.
When two string are compared internally what is checked is the unicode standard value that represents the caracter.

Uses a position based on the mentioned string system thus is divided in the 2 segments called n and z.

Segment n

The purpose for this segment is to move up or down in a string.

Number System

To achieve the goal to get use of the comparison system, a number system is needs to be created with the following axioms:

  • No negative number exists
  • The set contains at least 3 elements
  • Exits a final number sf
  • Exits a start number s0
  • Exits a zero number z
  • Exits a middle character m that is the value between sf and s0 such that m = ceil((sf + s0) / 2)

In the following example it is shown how to set a system using the 26 basic letters from the english alphabet [ a - z ] plus the exclamation mark [ ! ] as the zero.

As long as "zero < first < last" any values can be used
